react-bind-this

如果你使用了严格模式，那么没有显式的使用调用者 的情况下， this 永远不会自动绑定到全局对象上。如果此时你调用 greetCopy ，你就会得到报错，因为这时候 this不指向任何对象，this 这时候就是 undefined

如果我想使用另一个对象作为调用者来调用 fish.greet ，我该怎么做？这时候就要用到 Function.prototype.call 。

在调用的时候并没有显式的调用者，所以此时，this 在非严格模式下就会指向全局对象，在严格模式下就会指向 undefined 。

如果你使用了 ES6 的 class 语法，所有在 class 中声明的方法都会自动地使用严格模式

当你使用 onClick={this.handleClick}来绑定事件监听函数的时候，handleClick 函数实际上会作为回调函数，传入 addEventListener() 。这就是为什么你在 React 的组件中添加事件处理函数为什么会得到 undefnied 而不是全局对象或者别的什么东西。

箭头函数 this 永远绑定了定义箭头函数所在的那个对象

this指向全局变量的情况一般是： ‘This’ inside a function which is inside another function will be attached to the global object.

Qury工作总结-(半年)

Qury work summary

  1. Optimized existed PhantomJS based online rendering system

    • Previous performace: %50 <60s %50 >60s =》 %70 10s, %90 20s, 10% 60s
    • Communication mechanism
    • Task Queue: enable cancelling tasks
    • Rendering logic: split the previous logic to multiple steps and add corner case handling for each steps to improve the result quality(i.e. Blank white page, timeout)
    • Embedded no-result page detection and quality improvement feature.

  2. Maintained and rewrite the result rendering code (Android)

    • Figure out the UI drawing logic: Y = padding * (2^(i - scroll/height) - 1)
    • Cutomized adapter to control the UI
    • Cutomized webview to make the rendering logic at client side complete

  3. Desigined and developed a new real-time rendering system based on Selenium

    • Improve the performance of rendering (70% 5s, 95% 10s, 5% 60s)
    • Desigined rule based quality improvement method to improve the snapshot quality (no popup, no header, no ads, show content only)
    • Desigined IR, ML and Rule based model to detect no-result page (90% precision and 99% recall)

  4. Data crawling task of Instagram

    • get trending hashtags and posts recursively
    • crawl trending posts and get all hashtags of these trending posts
    • recursively using these newly crawled hashtags to get more related ones.

  5. Automation system to add new websites and apps for search and browsing.

  6. A web-based online file and configuration distribution system (config-hub).

    • Config ssh connection of servers within a cluster and use one of them to host website, enable various of operations within the cluster based on SSH.
    • Executing ssh commands or files.
    • Upload files to specific server or a cluster on GCP.
    • System level or Project level configuration (edit and apply)
    • Enable start and stop of machines

Detect blank snapshot

Backgroud:

A new model for blank page detection. <img src="/images/camila.jpeg" width="850"> <!-- more -->

Solution:

Based on the html, IR based detection is built. However, the recall is around 60% in order to maintain a high precision(>90%)

During the process of building IR detection model, we find that the 'no result' remind provided by different websites could be a strong signal. As a result, we create a config file to record the 'no result' string on different sites and do detection by matching the string. With this rules based model, the precision is improved to 95% and recall is 85%.

Update database automatically daily

Backgroud:

The blank page detection system need to use the demo blank page to compare the difference between a new search result and demo blank page. It is used as a signal for detection. However, the demo blank page could change as time goes by, so how to keep the demo blank page data updated automatically. <!-- more -->

Possible soultion:

  1. Database: mongoDB or Hbase. How to keep historical data?
  2. Trigger of execution: Time.sleep or Task scheduler
