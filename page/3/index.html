Detect blank snapshot

Backgroud:

The current snapshot server will always return rendered result without checking whether the snapshot has content or not, which hurt the user experience a lot. As a result, detecting blank snapshot is necessary to do. <img src="/images/IMG_2844.JPG" width="850"> <!-- more -->

Solution:

  1. Count word frequency and use the keywords frequency as a signal to detect. Easy to do, but low precision since threshhold is hard to decision and may varies app by app. Also, some apps just do not return search results by keywords matching.

  2. Page differece between demo blank page and the current page. Higher precision, but need to get demo blank page

Generate Snapshot for URL in real time

Backgroud:

Phantom JS was chosen as the core of snapshot server (phantom-cluster + phantom JS + node). <!-- more -->

Solutions:

  1. The success rate of snapshot server is low. Original system just wait until the server return snapshots but it may block.

As a result, timeout, cancellation and retry are added. When the execution time of a request is too long, cancel it and maybe retry.

  1. Configuration and code logic not good, which slow down the system.

Improve the configuration and re-design the logic. Remove unnecessary code and logic.

  1. Single powerful server(6 CPUs) is used but the phantom service is not good at parallel.

Use multiple single CPU servers instead. The cost is basically the same but performance improved a lot.
