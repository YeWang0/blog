react-bind-this

如果你使用了严格模式，那么没有显式的使用调用者 的情况下， this 永远不会自动绑定到全局对象上。如果此时你调用 greetCopy ，你就会得到报错，因为这时候 this不指向任何对象，this 这时候就是 undefined

如果我想使用另一个对象作为调用者来调用 fish.greet ，我该怎么做？这时候就要用到 Function.prototype.call 。

在调用的时候并没有显式的调用者，所以此时，this 在非严格模式下就会指向全局对象，在严格模式下就会指向 undefined 。

如果你使用了 ES6 的 class 语法，所有在 class 中声明的方法都会自动地使用严格模式

当你使用 onClick={this.handleClick}来绑定事件监听函数的时候，handleClick 函数实际上会作为回调函数，传入 addEventListener() 。这就是为什么你在 React 的组件中添加事件处理函数为什么会得到 undefnied 而不是全局对象或者别的什么东西。

箭头函数 this 永远绑定了定义箭头函数所在的那个对象

this指向全局变量的情况一般是：
‘This’ inside a function which is inside another function will be attached to the global object.
