Qury工作总结-(半年)

收获

一转眼，已经在Qury工作半年了，作为我毕业后的第一份工作，我还是非常满意的。

  1. CEO 和 CTO 人都很好，对我也很好。
  2. 工作很充实，也充满了挑战，让我学到了很多，见识了很多，也成熟了很多(技术 和 做人)。
  3. 对于工作中遇到的问题，我学习了如何去分析问题并解决问题，比以前更加稳重和可靠。
  4. 明白了时间的可贵，需要珍惜，去做有意义的事情。
  5. 代码能力有所提高。
  6. 意识到系统设计的重要性，需要在下一阶段中着重提高这方面的能力（分析，模块化，抽象，分层）
  7. 近距离的跟曾经了不起的人接触，看到了他们的优点和优势，也看到了他们的真实和不足。
  8. 优点一：勇气，敢于去交涉，遇事不退缩，以及一份自信。（Both）
  9. 优点二：做事靠谱，守信用，不怕吃苦，能够坚持。(CEO)
  10. 优点三：较高的智商 和 思维能力，分析问题的角度，善于学习。(CEO)
  11. 优点四：对于技术，有较高的要求和追求，代码能力和架构能力强（CTO）
  12. 优点五：克制，能够控制自己的情绪 和 欲望，做有利于自己的事，而不是一时兴起。(CEO)
  13. 优点六：为人正直，不偷奸耍滑，不钻空子。（Both）
  14. 优点七：思路清晰，目标明确，善于控场。（CEO）
  15. 缺点一：CEO代码能力不够强，写代码比较乱，可读性不高，且设计模式较弱，对于技术没有钻研精神。
  16. 缺点二：CEO社交能力不强，偏技术，不太会说话。
  17. 缺点三：CTO技术较强，智商较高但不断学习的能力不强。
  18. 缺点四：CTO自我控制能力较差，需要外部督促 或者 其他因素的激励。

工作成果

  1. Optimized existed PhantomJS based online rendering system
    • Previous performace: %50 <60s %50="">60s =》 %70 10s, %90 20s, 10% 60s
    • Communication mechanism
    • Task Queue: enable cancelling tasks
    • Rendering logic: split the previous logic to multiple steps and add corner case handling for each steps to improve the result quality(i.e. Blank white page, timeout)
    • Embedded no-result page detection and quality improvement feature.
  2. Maintained and rewrite the result rendering code (Android)
    • Figure out the UI drawing logic: Y = padding * (2^(i - scroll/height) - 1)
    • Cutomized adapter to control the UI
    • Cutomized webview to make the rendering logic at client side complete
  3. Desigined and developed a new real-time rendering system based on Selenium
    • Improve the performance of rendering (70% 5s, 95% 10s, 5% 60s)
    • Desigined rule based quality improvement method to improve the snapshot quality (no popup, no header, no ads, show content only)
    • Desigined IR, ML and Rule based model to detect no-result page (90% precision and 99% recall)

  4. Data crawling task of Instagram

    • get trending hashtags and posts recursively
    • crawl trending posts and get all hashtags of these trending posts
    • recursively using these newly crawled hashtags to get more related ones.

  5. Automation system to add new websites and apps for search and browsing.

    • prepare seed hashtags

  6. A web-based online file and configuration distribution system (config-hub).

    • Config ssh connection of servers within a cluster and use one of them to host website, enable various of operations within the cluster based on SSH.
    • Executing ssh commands or files.
    • Upload files to specific server or a cluster on GCP.
    • System level or Project level configuration (edit and apply)
    • Enable start and stop of machines
