Design Pattern - Simple Factory

Simple Factory Pattern. This allows interfaces for creating objects without exposing the object creation logic to the client.

Example

Design a calculator

Different operations i.e. +, -, *, /, ^ could be considered as subclasses of operation abstract class

Operation factory return a specific operation for you to use.

Same input, different output based on the opeation type.
